# Prompt Sidebar

Prompt Sidebar 是一款轻量级、高效率的浏览器扩展（Chrome Extension），旨在帮助用户在浏览器右侧以常驻侧边栏或快捷唤出的方式，集中管理和使用各种提示词（Prompt）。通过智能识别当前网页的文本框，用户可以实现一键应用提示词，彻底告别繁琐的“复制-粘贴”流程。

---

## ✨ 核心功能介绍

- **🚀 即时调用与一键填入**：无需离开当前页面，一键呼出侧边栏。智能识别当前网页聚焦的文本框，点击“应用”即可自动填入提示词。
- **📚 提示词高效管理**：支持提示词的新增、编辑、删除和收藏；内置支持按分类、标签进行横向胶囊筛选；通过搜索框实时检索（标题、内容、标签）。
- **📱 现代且丝滑的 UI 体验**：采用手风琴式（动态展开）阅读模式，点击卡片原地展开全文；内置虚拟滚动，轻松支撑 1000+ 条目而不卡顿。
- **☁️ 跨设备账号同步**：基于 Chrome Storage Sync，只需登录 Google 账号即可在多台设备间自动同步个人提示词库。
- **⚙️ 丰富的个性化设置**：支持自定义全局快捷键（默认 `Alt+Q`）、上下文悬浮球开关、黑名单网站配置，以及全量数据的 JSON 导入/导出备份。

## 🛠 开发与运行环境要求

### 开发环境依赖

- **Node.js**: v18.0.0 或更高版本
- **包管理器**: npm (v9+) 或 pnpm
- **开发工具**: 推荐使用 VS Code (配置 TypeScript、Tailwind CSS 和 ESLint 插件)

### 运行环境要求

- **浏览器**: Google Chrome (v100+) 或 Microsoft Edge (v100+)
- **扩展规范**: Manifest V3

## 🚀 项目部署与启动指引

### 1. 本地开发运行

1. 克隆或下载本项目到本地。
2. 在项目根目录下打开终端，安装依赖：
   ```bash
   npm install
   ```
3. 启动开发服务器（支持热更新）：
   ```bash
   npm run dev
   ```
4. 加载到浏览器：
   - 打开 Chrome 的扩展程序页面 (`chrome://extensions/`)
   - 开启右上角的 **开发者模式**
   - 点击 **加载已解压的扩展程序**，选择项目根目录下的 `dist` 文件夹。

### 2. 生产环境构建

1. 运行构建命令：
   ```bash
   npm run build
   ```
2. 构建完成后，`dist` 目录即为可分发的扩展程序包。
3. 按照上述步骤将 `dist` 文件夹加载到浏览器中即可使用。

## 🧩 核心代码模块说明

1. **`src/sidepanel` (侧边栏 UI)**：扩展的核心界面，基于 React 构建。负责渲染提示词列表、搜索框、分类胶囊栏，并处理卡片的手风琴式展开及按钮交互。
2. **`src/content-script` (网页注入脚本)**：自动注入到所有匹配的网页中。监听页面的 `focus` 事件以锁定当前活跃的文本输入框（支持 Shadow DOM），并在接收到侧边栏的“应用”消息时，将指定的提示词文本精准插入光标处。
3. **`src/background` (Service Worker)**：扩展的后台驻留脚本。负责监听快捷键 (`Alt+Q`) 触发、处理扩展图标的点击事件（打开侧边栏），并在扩展安装时执行初始化的通信中转。
4. **`src/store` (状态管理与存储)**：
   - `promptStore`: 基于 Zustand，管理所有提示词数据的增删改查。
   - `settingsStore`: 管理快捷键、悬浮球、同步开关等偏好设置。
   - `storage.ts`: 封装 `chrome.storage.local` 和 `chrome.storage.sync`，实现本地存储与跨设备云端同步的无缝切换。
5. **`src/options` (设置面板)**：提供用户界面的配置中心，支持数据导出备份（JSON格式）、重载内置 `prompt.md` 提示词库等高阶功能。

## 🔧 常见问题排查方案

| 问题现象                                    | 可能原因                             | 解决排查方案                                                                                                                          |
| ------------------------------------------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------- |
| **点击“应用”没有填入网页**          | 1. 网页未刷新 `<br>`2. 焦点未锁定  | 安装或更新插件后，请**必须刷新目标网页** (`F5`)，并用鼠标主动点击一下网页内的输入框（直到右下角出现悬浮球），再点击“应用”。 |
| **导出的 JSON 文件 `prompts` 为空** | 旧版状态同步延迟 bug                 | 请确保插件版本升级到 `v0.1.7` 或以上，新版本已强制要求导出前初始化数据缓存。                                                        |
| **快捷键冲突无法唤出**                | 浏览器或其他应用占用了 `Alt+Q`     | 进入插件设置页，在“快捷键”区域点击录制，更换一个不冲突的组合键（如 `Alt+P`）。                                                    |
| **开启同步后数据未更新**              | 浏览器未登录 Google 账号，或容量超限 | 确保 Chrome 已登录账号并开启系统级同步。Chrome Sync 单条数据容量限制为 8KB，总量 100KB，建议定期清理废弃提示词。                      |
| **页面右下角悬浮球遮挡内容**          | 悬浮球逻辑生效于全局                 | 如果觉得打扰，可以在设置页面关闭“启用悬浮球”选项，或者将该网页加入黑名单。                                                          |
